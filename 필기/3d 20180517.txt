리소스 상태

- 리소스들은 상태를 가지고 있다
- 서술자가 파이프라인과 연결을 시켜줌
- 한 순간의 하나의 상태를 가지도록 되어있음
- 상태는 읽기, 읽기/쓰기 로 나뉨
- GPU -> RENDERTARGET 그림을 그리기 위해서 사용
- 리소스의 상태 전이 는 프로그래머가 명시적으로 해주어야한다
- 렌더링 초기에 바꿔주는 일들이 있어야 한다
- 뎁스스텐실은 바꿔줄필요없음(시작할때하나로 설정해주면됨)

리소스 장벽
- 리소스의 암시적인 상태전이(동기화가 필요 없음)
- COMMON 으로 설정 되있는 상태는 자동으로 전이가 될 수 있음
- D3D12_RESOURCE_BARRIER_TYPE_TRANSITION // 리소스 전이를 나타내는 전이 장벽

명령 리스트
- 커맨드 리스트를 큐를 넣는다고 해서 실행이 되는것이 아니다
- 이걸 GPU가 꺼내가서 실행을 해야함
- 디바이스를 통해 직접 해야한다(그 즉시 실행됨)
- 메모리는 비디오 메모리에 할당됨
